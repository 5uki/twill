# Type Safety

> Tauri / Rust 层的类型安全重点在于：命令契约清晰、序列化可控、错误模型稳定。

---

## 基本规则

- 优先使用结构化类型表达命令输入输出，不用“字符串拼协议”。
- 能用 `struct` / `enum` 表达的约束，不要留给调用方猜。
- 跨层字段名和语义要统一，不做隐式重命名和隐式缺省。

---

## 序列化边界

- 命令入参与返回值必须显式定义序列化结构。
- 边界层负责格式转换，领域层尽量保持语义类型。
- 不把前端展示字段直接渗透进底层领域模型。

---

## 错误模型

- 错误类型要能区分输入错误、业务错误、系统错误。
- 不要把所有失败都压成同一个字符串。
- 如果前端需要针对错误类型做不同处理，后端必须提供可判断的错误信息。

---

## Forbidden Patterns

- 过度使用裸 `String`、`serde_json::Value` 作为核心业务输入输出
- 无理由的 `unwrap()` / `expect()` 进入生产路径
- 为了图省事把多个不同语义字段塞进一个松散对象
