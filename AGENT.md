# Twill Antigravity IDE Instructions

## 会话启动规则

- 新会话默认先使用 `/trellis:start`。
- 如果没有先使用 `/trellis:start`，不要直接开始开发任务。
- 应先询问用户是否明确选择跳过 `/trellis:start`。
- 如果用户选择跳过，仍需先补做最小上下文检查：
  - 阅读 `.trellis/workflow.md`
  - 阅读相关 `.trellis/spec/*/index.md`
  - 确认当前任务、改动范围和上下文

## 项目上下文

- Twill 是基于 `Tauri + React + TypeScript + Rust` 的多端邮箱客户端应用。
- UI 框架明确是 `React`。
- `src/` 是 React UI 层。
- `src-tauri/` 是 Tauri / Rust 原生层。

## 硬性规则

- 所有思考、分析、解释和回答默认使用简体中文。
- 代码注释、技术文档、API 文档和用户说明默认使用中文。
- 严格遵循 TDD：`Red -> Green -> Refactor`。
- 所有功能都必须提供 CLI 模拟器。
- 没有 CLI 模拟器的功能，默认视为未完成。
- CLI、Tauri 命令和测试必须共用同一套核心应用逻辑。
- 修改功能时不默认保留旧兼容代码，除非需求明确要求。
- 不做过度防御性编程，优先保持实现简洁、清晰、可维护。

## 质量关注点

- 关注时间复杂度、空间复杂度和热点路径。
- 关注内存、IO 和系统调用成本。
- 覆盖成功路径、失败路径和关键边界场景。
- 跨层改动必须验证命令契约、类型映射和 CLI 覆盖。

## 输出风格

- 面向终端阅读优化输出。
- 复杂问题先给核心结论，再展开。
- 使用清晰分组、短列表和必要留白。
- 复杂流程优先使用简洁 ASCII 图示。
- 多行代码、配置和日志使用带语言标识的 Markdown 代码块。
